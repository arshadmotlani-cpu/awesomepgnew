# Financial Audit Report

> Generated by `scripts/audit-financials.ts` — full customer scan with 8 integrity checks.

## Checks

| Code | Description | Auto-repair |
|------|-------------|-------------|
| `DEPOSIT_SHORTFALL_NOT_INVOICED` | Expected deposit vs collected; shortfall not on open invoice | Yes — creates deposit invoice |
| `INVOICE_EMPTY` | Non-cancelled invoice with amount but empty `breakdown.lines` | Manual review |
| `INVOICE_TOTAL_MISMATCH` | `amountPaise` ≠ sum of `breakdown.lines` | Yes — fixes `amountPaise` from lines |
| `PAYMENT_NOT_RECONCILED` | Succeeded payment but invoice still open when covered | Yes — sets paid/partial |
| `DEPOSIT_LEDGER_NEGATIVE` | Net deposit ledger balance negative | Manual review (append-only ledger) |
| `MISSING_RENT_INVOICE` | Active booking, billing month started, no rent invoice | Manual review |
| `DUPLICATE_INVOICE` | Duplicate non-cancelled for same booking+month+type | Manual review |
| `OUTSTANDING_NOT_SURFACED` | Engine outstanding > visible open invoice balances | Manual review |

## Usage

```bash
# Read-only scan — writes audit-output-{timestamp}.json
npx tsx scripts/audit-financials.ts

# Dry-run repairs
npx tsx scripts/repair-financials.ts --dry-run

# Apply safe repairs
npx tsx scripts/repair-financials.ts
```

## Daily cron

- Route: `GET/POST /api/cron/financial-reconciliation` (Bearer `CRON_SECRET`)
- Schedule: `30 6 * * *` UTC (see `vercel.json`)
- Logs: `audit_log` with `action = daily_reconciliation`
- Manual issues → `action_items` for admin Action Center

## SSOT

- Audit logic: `src/services/financialIntegrityAudit.ts`
- Repairs: `src/services/financialRepair.ts` (deposit ledger append-only)
- Reconciliation: `src/services/financialReconciliation.ts`
- Outstanding balance: `getLiveOutstandingBalance()` / `getResidentFinancialSummary()`

## Local run

Run against your local DB with `.env` configured. If no DB is available, use unit tests:

```bash
npm test -- tests/unit/financialAudit.test.ts tests/unit/financialRepair.test.ts tests/unit/financialReconciliation.test.ts
```

Dry-run against production is **not** recommended without `--dry-run` first.
